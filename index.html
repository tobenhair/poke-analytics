Track. Analyse. Decide.

A personal investment dashboard for sealed Pokémon TCG products. Track prices and set values over time, surface buying signals, and explore scenarios — all from a single Excel file you control.

📊
Analysis

The decision-making view. See all your tracked products ranked by weighted score, compare set value per booster, watch price history per product, and explore scenarios.

  • KPIs at a glance — top score, best value, newest release
  • Filter by Booster Box / ETB / Bundle and sort by any metric
  • Buy-signal flags 💰 when price drops but set value holds
  • Scenario explorer — drag sliders to test outcomes
  • Adjustable age threshold for weighted scoring
✏️
Data Entry

The monthly update view. Enter the latest Price and Set Value for each product, add new releases, attach Cardmarket links, then export the updated file.

  • One row per product — last known values shown for reference
  • Today's date pre-filled as the snapshot label
  • Add new products with name, type, release date, initial price
  • Attach & edit Cardmarket URLs per product 🔗
  • Export updated .xlsx ready to commit back to the repo
How it works
1 Load

pokemon_data.xlsx auto-loads from the repo on every visit, or drag-drop your own file onto the Analysis tab.

2 Update

Once a month, fetch latest prices from Cardmarket and enter them in the Data Entry tab. New products can be added at any time.

3 Decide

Use the Analysis tab to spot buy signals, compare options, and check the weighted score before pulling the trigger.

Key concepts
Glossary
Set Value The total market value of all cards in a complete set.
Price / Booster The product price ÷ number of boosters inside (BOX = 36, ETB = 9, BUNDLE = 6).
SV / Booster Set Value ÷ Price/Booster. The core comparability metric — works equally well across BOX, ETB, and BUNDLE products.
Age Weight A penalty applied to newer products to reflect that they haven't had time to appreciate. Adjustable via the Age Threshold slider.
Wtd. Score SV / Booster × Age Weight. The primary ranking metric used by the dashboard.
💰 Buy Signal Flagged when a product's price has dropped at least 5% in the last snapshot, while its set value held within ±5% — a possible mispricing.
Built for personal use. Always do your own research. Past performance does not guarantee future results.

📋 File Format Guide

Your .xlsx file must contain exactly two sheets named Summary and Historical Data. Column names must match exactly (case-sensitive).

Sheet 1 — Summary

One row per product. Each row becomes an entry in the main table.

Column nameRequired?Format / Notes
ProductrequiredFull product name — must also appear in Historical Data sheet
TyperequiredExactly one of: BOX, ETB, or BUNDLE
Release DaterequiredDate value or text in YYYY-MM-DD format
Age (years)requiredDecimal number, e.g. 1.47
Current Price (€)requiredNumber in cents or euros — must be consistent across rows
Set Value (€)requiredTotal set value — same unit as price
Price / Booster (€)requiredCurrent price divided by number of boosters in the product
SV / BoosterrequiredSet value divided by Price / Booster
Age WeightrequiredDecimal 0–1. Products <1yr are penalised; ≥3yr = 1.0
Wtd. ScorerequiredSV / Booster × Age Weight

Sheet 2 — Historical Data

One row per product per snapshot. Add a new row each month when you update prices.

Column nameRequired?Format / Notes
ProductrequiredMust exactly match the Product name in the Summary sheet
Snapshot DaterequiredDate label shown on chart axes, must be in YYYY-MM-DD format (ISO 8601), e.g. 2026-04-20
Price (€)requiredProduct price at that snapshot. Leave blank if not yet tracked.
Set Value (€)requiredSet value at that snapshot. Leave blank if not yet tracked.
Monthly workflow: Each month, add a new column to your Historical Data sheet with that month's date label, then fill in the latest Price and Set Value for each product. Re-save the file and drop it onto the dashboard — all charts and signals will update automatically.

New products: Add a new row to Summary, then add matching rows to Historical Data. The product name must match exactly in both sheets.
